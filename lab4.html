    Лабораторная работа №4 — Адаптивная и семантически корректная вёрстка интерфейса

    Цель

    Сверстать профессиональную адаптивную страницу интерфейса (дашборд / каталог / сервис) по индивидуальной предметной области (сервис №1–30), используя:

    • семантическую HTML-структуру;
    • mobile-first подход;
    • Flexbox + Grid для компоновки;
    • breakpoints через @media (min-width: …);
    • sticky header;
    • responsive navigation (burger menu);
    • адаптивные карточки;
    • clamp() для типографики;
    • проверку Lighthouse (Performance + Accessibility).

    1. Теория

    1.1. Семантически корректная вёрстка интерфейса

    Семантика = правильные теги под смысл:

    • <header> — шапка/навигация;
    • <nav> — меню;
    • <main> — уникальный контент страницы;
    • <section> — логические разделы;
    • <article> — самостоятельная карточка/единица контента;
    • <aside> — вспомогательные блоки (фильтры, подсказки);
    • <footer> — подвал;
    • aria-* — доступность интерактивных элементов.

    Пример каркаса страницы

    <header class="header" id="top">
  <a class="brand" href="#top">MyService</a>

  <button class="burger" type="button"
          aria-label="Открыть меню"
          aria-controls="nav"
          aria-expanded="false">
    ☰
  </button>

  <nav class="nav" id="nav" aria-label="Основная навигация">
    <a href="#catalog">Каталог</a>
    <a href="#filters">Фильтры</a>
    <a href="#stats">Статистика</a>
  </nav>
</header>

<main id="main">
  <section id="catalog" aria-label="Каталог">
    <h1>Каталог</h1>

    <div class="grid">
      <article class="card">
        <h2>Карточка</h2>
        <p>Описание элемента</p>
      </article>
    </div>
  </section>

  <aside id="filters" aria-label="Фильтры">
    <h2>Фильтры</h2>
  </aside>
</main>

<footer>
  <small>© 2026 Учебный проект</small>
</footer>

    1.2. Mobile-first подход

    Mobile-first означает:

    • базовые стили — для маленького экрана;
    • расширение — через @media (min-width: …).

    Пример mobile-first (карточки)

    .grid { display: grid; gap: 12px; grid-template-columns: 1fr; }

@media (min-width: 768px) {
  .grid { grid-template-columns: repeat(2, 1fr); }
}

@media (min-width: 1024px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
}

    1.3. Flexbox и Grid в интерфейсе

    • Flexbox — для “линейных” блоков (header, toolbar, кнопки).
    • Grid — для сеток (каталог, карточки, dashboard).

    Пример: header на Flexbox

    .header{
  display:flex;
  align-items:center;
  justify-content:space-between;
  gap:12px;
}

    Пример: каталог на Grid

    .grid{
  display:grid;
  grid-template-columns:1fr;
  gap:16px;
}

    1.4. Breakpoints (min-width): как выбрать

    Типовые контрольные точки:

    • 480px — маленькие смартфоны;
    • 768px — планшеты;
    • 1024px — ноутбуки;
    • 1280px — большие экраны.

    Важно: не “под устройства”, а под контент: когда интерфейс “ломается”.

    Пример набора breakpoint-ов

    @media (min-width: 480px) { /* мелкие улучшения */ }
@media (min-width: 768px) { /* 2 колонки */ }
@media (min-width: 1024px){ /* 3 колонки + сайдбар */ }

    1.5. Sticky header

    Sticky header — шапка “прилипает” сверху при прокрутке.

    Пример sticky

    .header{
  position: sticky;
  top: 0;
  z-index: 10;
  background: white;
  border-bottom: 1px solid #ddd;
}

    1.6. Responsive navigation (burger menu)

    Бургер-меню:

    • на мобильных скрывает навигацию;
    • по кнопке раскрывает;
    • aria-expanded меняется для доступности.

    Пример CSS: меню скрыто на mobile

    .nav{ display:none; }
.nav.is-open{ display:flex; flex-direction:column; gap:8px; }

@media (min-width: 768px){
  .burger{ display:none; }
  .nav{ display:flex; flex-direction:row; gap:16px; }
}

    Пример JS: переключение меню

    <script>
  const burger = document.querySelector('.burger');
  const nav = document.querySelector('#nav');

  burger.addEventListener('click', () => {
    const opened = nav.classList.toggle('is-open');
    burger.setAttribute('aria-expanded', String(opened));
  });
</script>

    1.7. Layout карточек (карточный интерфейс)

    Карточка должна быть:

    • адаптивной по высоте;
    • с читабельной типографикой;
    • с кнопками/бейджами внутри.

    Пример карточки

    .card{
  border:1px solid #ddd;
  border-radius:16px;
  padding:16px;
  display:flex;
  flex-direction:column;
  gap:10px;
}
.card__actions{
  margin-top:auto;
  display:flex;
  gap:8px;
}

    1.8. clamp() для адаптивной типографики

    clamp(min, preferred, max) позволяет сделать текст адаптивным.

    Пример

    h1{ font-size: clamp(1.25rem, 2vw + 1rem, 2rem); }
p{ font-size: clamp(0.95rem, 0.6vw + 0.8rem, 1.1rem); }

    1.9. Lighthouse: Performance + Accessibility

    Lighthouse оценивает:

    • производительность (тяжёлые изображения, блокировки);
    • доступность (label, aria, контраст);
    • best practices.

    Минимальные требования для хороших баллов

    • alt у изображений;
    • label у полей;
    • button с aria-label;
    • видимый focus;
    • не использовать огромные изображения без нужды.

    2. Задания

    Страница — адаптивный интерфейс под предметную область (дашборд / каталог / сервис).

    2.1. Выбор предметной области и типа интерфейса

    1. Выберите тип страницы: дашборд или каталог или сервисная страница под свою предметную область.
    2. Запишите в отчёте: сервис, цель интерфейса, пользователь.

    2.2. Семантический каркас

    Сверстайте каркас:

    • <header> + <nav>
    • <main>
    • минимум 2 <section>
    • минимум 6 <article> (карточки)
    • <aside> (фильтры/подсказки)
    • <footer>

    2.3. Mobile-first стили

    1. Базовая верстка под 320–480px.
    2. Использовать @media (min-width: …) минимум 2 раза.

    2.4. Flexbox и Grid

    • header и панель действий — на Flexbox;
    • каталог/карточки — на Grid.

    2.5. Sticky header

    Шапка должна оставаться сверху при прокрутке.

    2.6. Responsive navigation

    • на мобильных — burger menu (кнопка);
    • на планшете/десктопе — обычное меню.

    2.7. Layout карточек

    Карточки должны:

    • аккуратно растягиваться в сетке,
    • иметь заголовок, описание и действия,
    • не “ломаться” на разных ширинах.

    2.8. clamp() для типографики

    Использовать clamp() минимум:

    • для h1,
    • для базового текста (p или .text).
