    Лабораторная работа №5 — Интерактивность интерфейса с использованием JavaScript

    Цель

    Разработать интерактивный пользовательский интерфейс по индивидуальной предметной области (сервис №1–30), используя чистый JavaScript без фреймворков, реализовав:

    • DOM-manipulation;
    • обработку событий;
    • toggle-классы;
    • модальное окно;
    • Tabs или Accordion;
    • фильтрацию списка;
    • динамическое добавление элементов;
    • работу с localStorage;
    • debounce для поиска.

    1. Теория

    1.1. Что такое JavaScript и зачем он нужен

    JavaScript (JS) — это язык программирования, который выполняется в браузере и управляет поведением веб-страницы.

    Если рассматривать веб-технологии системно:

    • HTML — структура страницы (что размещено);
    • CSS — оформление (как это выглядит);
    • JavaScript — логика и поведение (что происходит).

    Логическая связка:

    • HTML отвечает на вопрос «что есть?»
    • CSS отвечает на вопрос «как выглядит?»
    • JS отвечает на вопрос «что делать?»

    Без JavaScript сайт остаётся статичным. С JavaScript страница превращается в интерактивное web-приложение.

    1.2. Переменные и типы данных

    Любая программа работает с данными. Чтобы их использовать, необходимо сохранить их в памяти.

    Переменная — это именованная область памяти для хранения значения.

    let name = "Анна";
let age = 20;
let isActive = true;

    Способы объявления переменных

    • let — значение можно изменять;
    • const — значение изменять нельзя;
    • var — устаревший способ (не рекомендуется).
    let count = 0;
count = 5; // изменение допустимо

const PI = 3.14;
// PI = 3.15; ❌ ошибка

    Логика использования:

    • Если значение должно изменяться — используем let.
    • Если значение фиксированное — используем const.
    • В профессиональной разработке по умолчанию применяют const, повышая надёжность кода.

    Основные типы данных

    • string — строка
    • number — число
    • boolean — логическое значение (true/false)
    • array — массив
    • object — объект
    • null
    • undefined

    1.3. Алгоритмические конструкции

    Программа — это последовательность действий (алгоритм).

    Условные конструкции

    if (age >= 18) {
  console.log("Доступ разрешён");
} else {
  console.log("Доступ запрещён");
}

    Логика:

    1. Получаем данные.
    2. Проверяем условие.
    3. Выполняем нужное действие.

    Такая логика используется в:

    • проверке форм;
    • авторизации;
    • фильтрации данных;
    • управлении доступом.

    1.4. Циклы

    Цикл — это повторение действия.

    Цикл for

    for (let i = 0; i < 5; i++) {
  console.log(i);
}

    Цикл while

    let i = 0;
while (i < 5) {
  console.log(i);
  i++;
}

    Перебор массива

    const arr = [1, 2, 3];

arr.forEach(item => {
  console.log(item);
});

    Логика:

    Циклы позволяют автоматизировать повторяющиеся действия, особенно при работе со списками данных или элементами интерфейса.

    1.5. Функции и передача параметров

    Функция — это блок кода, который можно вызывать многократно.

    function sum(a, b) {
  return a + b;
}

console.log(sum(2, 3));

    Логика функций:

    • разделяют код на независимые части;
    • уменьшают дублирование;
    • повышают читаемость;
    • позволяют повторно использовать алгоритмы.

    Передача параметров

    function greet(name) {
  console.log("Привет, " + name);
}

greet("Анна");

    Параметры делают функцию универсальной.

    Передача массива в функцию

    function printArray(arr) {
  arr.forEach(item => console.log(item));
}

const numbers = [10, 20, 30];
printArray(numbers);

    Массив передаётся по ссылке, поэтому его можно изменить внутри функции:

    function addItem(arr, value) {
  arr.push(value);
}

addItem(numbers, 40);

    1.6. Математические функции

    JavaScript содержит встроенный объект Math.

    Math.random();
Math.floor(4.7);
Math.ceil(4.2);
Math.round(4.5);
Math.max(1, 5, 3);
Math.min(1, 5, 3);

    Пример генерации случайного числа:

    const random = Math.floor(Math.random() * 10);

    Используются для:

    • расчётов;
    • округления цен;
    • генерации случайных значений;
    • игровой логики.

    1.7. DOM (Document Object Model)

    Когда браузер загружает HTML, он преобразует его в структуру объектов — DOM.

    JavaScript работает не с текстом HTML, а с DOM-объектами.

    Это позволяет:

    • изменять структуру страницы;
    • добавлять элементы;
    • удалять элементы;
    • управлять интерфейсом динамически.

    1.8. DOM-manipulation: управление структурой страницы

    DOM-manipulation — это изменение HTML-структуры через JavaScript.

    Возможности:

    • создавать элементы;
    • удалять элементы;
    • менять текст;
    • менять атрибуты;
    • менять классы;
    • перестраивать списки.

    Пример: изменить текст и класс

    const status = document.querySelector("#status");

status.textContent = "Активно";
status.classList.add("is-active");

    Логика:

    1. Находим элемент.
    2. Изменяем его состояние.
    3. Интерфейс обновляется.

    Пример: создать элемент карточки

    const card = document.createElement("article");
card.className = "card";

card.innerHTML = `
  <h3>Новая карточка</h3>
  <p>Добавлена через JS</p>
`;

document.querySelector("#list").append(card);

    Элемент создаётся динамически — он отсутствует в исходном HTML.

    1.9. Обработка событий: click, input, submit, keydown

    События — это реакция интерфейса на действия пользователя.

    Обработчик клика

    document.querySelector("#addBtn")
  .addEventListener("click", () => {
    console.log("Кнопка нажата");
  });

    Обработчик ввода

    document.querySelector("#search")
  .addEventListener("input", (e) => {
    console.log("Пользователь ввёл:", e.target.value);
  });

    Логика событий:

    1. Пользователь выполняет действие.
    2. Браузер генерирует событие.
    3. JavaScript выполняет обработчик.

    1.10. Toggle-классы: управление состоянием интерфейса

    Toggle-класс — механизм переключения состояния через CSS.

    const panel = document.querySelector("#panel");

document.querySelector("#toggleBtn")
  .addEventListener("click", () => {
    panel.classList.toggle("is-open");
  });

    JS управляет логикой состояния, CSS отвечает за визуальное отображение.

    1.11. Модальное окно: структура и логика

    Модальное окно — элемент интерфейса поверх страницы.

    Основные требования:

    • затемнение фона;
    • управление фокусом;
    • закрытие по кнопке;
    • закрытие по клавише Esc.

    HTML

    <button id="openModal">Открыть</button>

<div class="modal" id="modal" hidden aria-hidden="true">
  <div class="modal__content"
       role="dialog"
       aria-modal="true"
       aria-label="Детали">

    <button id="closeModal">×</button>
    <h2>Детали</h2>
    <p>Контент модального окна</p>

  </div>
</div>

    JS

    const modal = document.querySelector("#modal");
const open = document.querySelector("#openModal");
const close = document.querySelector("#closeModal");

function openModal() {
  modal.hidden = false;
  modal.setAttribute("aria-hidden", "false");
}

function closeModal() {
  modal.hidden = true;
  modal.setAttribute("aria-hidden", "true");
}

open.addEventListener("click", openModal);
close.addEventListener("click", closeModal);

document.addEventListener("keydown", (e) => {
  if (e.key === "Escape" && !modal.hidden) {
    closeModal();
  }
});

    1.12. Tabs и Accordion

    Оба механизма реализуют выбор состояния.

    Tabs

    Позволяют показать один активный раздел и скрыть остальные.

    Accordion

    Позволяет раскрывать и сворачивать секции.

    Логика в обоих случаях:

    • определить активный элемент;
    • скрыть остальные;
    • обновить атрибуты состояния.

    1.13. Фильтрация списка

    Фильтрация — это логическая обработка коллекции элементов.

    function applyFilter(category) {
  document.querySelectorAll(".item")
    .forEach(el => {
      const ok =
        category === "all" ||
        el.dataset.category === category;

      el.hidden = !ok;
    });
}

    Логика:

    1. Перебор элементов.
    2. Проверка условия.
    3. Скрытие неподходящих.

    1.14. Динамическое добавление элементов

    function addItem(title, category) {
  const el = document.createElement("article");

  el.className = "item";
  el.dataset.category = category;

  el.innerHTML = `
    <h3>${title}</h3>
    <p>Категория: ${category}</p>
  `;

  document.querySelector("#list").prepend(el);
}

    Используется при:

    • добавлении заявок;
    • создании карточек;
    • формировании списков.

    1.15. localStorage: сохранение состояния

    localStorage позволяет хранить данные между перезагрузками страницы.

    localStorage.setItem("filter", "electronics");

const saved = localStorage.getItem("filter") || "all";
applyFilter(saved);

    Для хранения сложных структур используется JSON:

    const items = JSON.parse(localStorage.getItem("items") || "[]");

items.push({ title: "Item", category: "x" });

localStorage.setItem("items", JSON.stringify(items));

    1.16. Debounce: оптимизация поиска

    Debounce — механизм отложенного выполнения функции.

    function debounce(fn, delay = 300) {
  let t = null;

  return (...args) => {
    clearTimeout(t);
    t = setTimeout(() => fn(...args), delay);
  };
}

    Использование:

    const onSearch = debounce((q) => {
  q = q.trim().toLowerCase();

  document.querySelectorAll(".item")
    .forEach(el => {
      const text = el.textContent.toLowerCase();
      el.hidden = !text.includes(q);
    });
}, 300);

document.querySelector("#search")
  .addEventListener("input", (e) => {
    onSearch(e.target.value);
  });

    Логика:

    • без debounce функция вызывается на каждую букву;
    • с debounce — только после паузы;
    • уменьшается нагрузка и повышается производительность.

    2. Задания

    2.1. Подготовка проекта

    Структура:

    lr5/
  index.html
  css/styles.css
  js/app.js
  report/

    2.2. DOM-manipulation

    Реализовать минимум 3 операции:

    • создание элемента (createElement);
    • изменение текста/атрибутов;
    • удаление элемента или замена контента.

    2.3. Обработка событий

    Использовать минимум 4 типа событий:

    • click,
    • input,
    • submit,
    • keydown (например Esc).

    2.4. Toggle-классы

    Реализовать минимум 2 переключаемых состояния:

    • “открыто/закрыто” (панель/фильтры/меню),
    • “выбрано/активно” (карточка/кнопка/таб).

    2.5. Модальное окно

    Модалка должна:

    • открываться по кнопке;

    • закрываться по:

    • кнопке “×”,

    • клику по overlay,
    • клавише Esc;
    • иметь role="dialog" и aria-modal="true".

    2.6. Tabs или Accordion (обязательно — выбрать 1)

    • Tabs: минимум 2 вкладки
    • Accordion: минимум 2 секции

    2.7. Фильтрация списка

    Список минимум 6 элементов (карточек/заявок/товаров). Фильтрация минимум по 2 критериям:

    • категория (select/radio),
    • поиск по тексту (input + debounce).

    2.8. Динамическое добавление элементов

    Сделать форму “Добавить элемент”:

    • ввод названия,
    • выбор категории,
    • добавление в список без перезагрузки.

    2.9. localStorage

    Сохранить минимум 2 состояния:

    • список добавленных элементов,
    • выбранный фильтр и/или строка поиска, и/или открытая вкладка.

    При перезагрузке страницы состояние должно восстанавливаться.

    2.10. Debounce для поиска

    Поиск должен использовать debounce (200–400 мс). В отчёте кратко объяснить зачем.
