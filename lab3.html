    Лабораторная работа №3 — Разработка дизайн-системы интерфейса с использованием CSS

    Цель

    Создать мини-дизайн-систему (UI-kit) для веб-интерфейса по индивидуальной предметной области (сервис №1–30), реализовав:

    • цветовую палитру (primary/secondary/accent/neutral);
    • типографическую шкалу (rem, line-height, scale);
    • CSS Custom Properties (переменные);
    • набор UI-компонентов: Button, Card, Input, Alert;
    • состояния hover/focus/active/disabled;
    • тёмную тему через prefers-color-scheme;
    • мини-гайд по использованию компонентов и демонстрационную страницу.

    1. Теория

    1.1. Дизайн-система vs “просто стили”

    Просто стили — это разрозненные правила “под конкретную страницу”. Дизайн-система — это набор стандартов, которые позволяют собирать интерфейс как конструктор:

    • единые цвета (tokens);
    • единая типографика;
    • единые отступы/радиусы/тени;
    • готовые компоненты (кнопки/карточки/поля);
    • предсказуемые состояния и доступность.

    Пример: “плохо” vs “нормально”

    Плохо: везде разные #-цвета, размеры, случайные px. Нормально: все значения берутся из переменных:

    :root {
  --color-primary: #2563eb;
  --radius-md: 12px;
  --space-3: 12px;
}
.button { background: var(--color-primary); border-radius: var(--radius-md); padding: var(--space-3); }

    1.2. Цветовая палитра: primary / secondary / accent / neutral

    Цвета в дизайн-системе задаются как токены (переменные). Минимально нужно 4 группы:

    • primary — основной (главная кнопка, ссылки);
    • secondary — вторичный (вторые кнопки/бейджи);
    • accent — акцент (особые действия/подсветка);
    • neutral — фон, текст, границы (UI-база).

    Пример палитры через CSS-переменные

    :root {
  /* primary / secondary / accent */
  --primary-600: #2563eb;
  --secondary-600: #16a34a;
  --accent-600: #dc2626;

  /* neutral */
  --neutral-0: #ffffff;
  --neutral-50: #f8fafc;
  --neutral-200: #e2e8f0;
  --neutral-700: #334155;
  --neutral-900: #0f172a;

  /* semantic (для компонентов) */
  --bg: var(--neutral-0);
  --surface: var(--neutral-50);
  --text: var(--neutral-900);
  --muted: var(--neutral-700);
  --border: var(--neutral-200);
}

    1.3. Типографическая шкала: rem, line-height, scale

    rem — единица, зависящая от html { font-size }. Это удобно для масштабирования. Шкала — набор размеров (например: 12/14/16/20/24/32).

    Пример типографики (шкала + line-height)

    :root{
  --font-sans: system-ui, -apple-system, "Segoe UI", Roboto, Arial, sans-serif;

  --text-xs: 0.75rem;  /* 12px */
  --text-sm: 0.875rem; /* 14px */
  --text-md: 1rem;     /* 16px */
  --text-lg: 1.25rem;  /* 20px */
  --text-xl: 1.5rem;   /* 24px */

  --lh-tight: 1.2;
  --lh-normal: 1.5;
  --lh-loose: 1.7;
}

body{
  font-family: var(--font-sans);
  font-size: var(--text-md);
  line-height: var(--lh-normal);
}
h1{ font-size: var(--text-xl); line-height: var(--lh-tight); }
h2{ font-size: var(--text-lg); line-height: var(--lh-tight); }
small{ font-size: var(--text-sm); }

    1.4. CSS Custom Properties: зачем и как применять

    CSS-переменные позволяют:

    • быстро менять тему;
    • поддерживать единый стиль;
    • уменьшить “магические числа”.

    Пример токенов размеров (spacing/radius/shadow)

    :root{
  --space-1: 4px;
  --space-2: 8px;
  --space-3: 12px;
  --space-4: 16px;
  --space-6: 24px;

  --radius-sm: 8px;
  --radius-md: 12px;
  --radius-lg: 16px;

  --shadow-sm: 0 2px 10px rgba(0,0,0,.08);
}
.card{
  padding: var(--space-4);
  border-radius: var(--radius-lg);
  box-shadow: var(--shadow-sm);
}

    1.5. Компонент Button: 3 состояния + варианты

    Кнопка — базовый компонент. Требование: минимум 3 состояния.

    Пример HTML

    <button class="btn btn--primary">Primary</button>
<button class="btn btn--secondary">Secondary</button>
<button class="btn btn--ghost">Ghost</button>

    Пример CSS (normal / hover / active + focus + disabled)

    .btn{
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: 8px;
  padding: 10px 14px;
  border-radius: var(--radius-md);
  border: 1px solid transparent;
  font: inherit;
  cursor: pointer;
  transition: transform .06s ease, background .15s ease, border-color .15s ease;
}

.btn:focus-visible{
  outline: 3px solid color-mix(in srgb, var(--primary-600) 30%, transparent);
  outline-offset: 2px;
}

.btn:active{ transform: translateY(1px); }
.btn:disabled{ opacity: .6; cursor: not-allowed; }

.btn--primary{
  background: var(--primary-600);
  color: white;
}
.btn--primary:hover{ background: color-mix(in srgb, var(--primary-600) 85%, black); }

.btn--secondary{
  background: var(--secondary-600);
  color: white;
}

.btn--ghost{
  background: transparent;
  border-color: var(--border);
  color: var(--text);
}
.btn--ghost:hover{ background: var(--surface); }

    1.6. Компонент Card: контейнер интерфейса

    Карточка собирает контент в блок.

    Пример HTML

    <article class="card">
  <h3 class="card__title">Карточка</h3>
  <p class="card__text">Описание предметной области или элемента.</p>
  <div class="card__actions">
    <button class="btn btn--primary">Действие</button>
    <button class="btn btn--ghost">Отмена</button>
  </div>
</article>

    Пример CSS

    .card{
  background: var(--bg);
  border: 1px solid var(--border);
  border-radius: var(--radius-lg);
  padding: var(--space-4);
  box-shadow: var(--shadow-sm);
}
.card__title{ margin: 0 0 var(--space-2); font-size: var(--text-lg); }
.card__text{ margin: 0 0 var(--space-4); color: var(--muted); }
.card__actions{ display: flex; gap: var(--space-2); }

    1.7. Компонент Input: поле + label + helper + error

    Поле ввода должно иметь состояния focus/invalid/disabled.

    Пример HTML

    <div class="field">
  <label class="label" for="q">Поиск</label>
  <input class="input" id="q" type="text" placeholder="Введите запрос">
  <small class="help">Подсказка: минимум 3 символа</small>
</div>

    Пример CSS (focus/invalid/disabled)

    .label{ display:block; margin-bottom: 6px; font-size: var(--text-sm); color: var(--muted); }

.input{
  width: 100%;
  padding: 10px 12px;
  border-radius: var(--radius-md);
  border: 1px solid var(--border);
  background: var(--bg);
  color: var(--text);
}

.input:focus-visible{
  outline: 3px solid color-mix(in srgb, var(--primary-600) 30%, transparent);
  border-color: color-mix(in srgb, var(--primary-600) 50%, var(--border));
}

.input[aria-invalid="true"]{
  border-color: var(--accent-600);
}

.input:disabled{ background: var(--surface); opacity: .7; }
.help{ color: var(--muted); font-size: var(--text-sm); }

    1.8. Компонент Alert: информация/успех/ошибка

    Alert показывает статус или сообщение системе.

    Пример HTML

    <div class="alert alert--info" role="status">
  <strong>Инфо:</strong> Заполните обязательные поля.
</div>

<div class="alert alert--error" role="alert">
  <strong>Ошибка:</strong> Неверный формат телефона.
</div>

    Пример CSS

    .alert{
  border: 1px solid var(--border);
  background: var(--surface);
  padding: var(--space-3);
  border-radius: var(--radius-md);
}
.alert--info{ border-color: color-mix(in srgb, var(--primary-600) 35%, var(--border)); }
.alert--error{ border-color: color-mix(in srgb, var(--accent-600) 45%, var(--border)); }

    1.9. Hover / Focus / Active: почему это обязательно

    Состояния нужны, чтобы интерфейс был предсказуемым:

    • hover: есть реакция на мышь
    • focus: доступно клавиатурой
    • active: подтверждение клика/нажатия

    Пример “общих” правил состояний

    a:focus-visible, button:focus-visible, input:focus-visible{
  outline: 3px solid color-mix(in srgb, var(--primary-600) 30%, transparent);
  outline-offset: 2px;
}

    1.10. Темная тема: prefers-color-scheme (без JS)

    Тёмная тема делается переопределением токенов в медиазапросе.

    Пример dark mode

    @media (prefers-color-scheme: dark) {
  :root{
    --bg: #0b1220;
    --surface: #0f172a;
    --text: #e5e7eb;
    --muted: #cbd5e1;
    --border: #243044;

    --shadow-sm: 0 2px 12px rgba(0,0,0,.35);
  }
}

    1.11. Мини-гайд по использованию компонентов (что писать в README)

    Мини-гайд — это инструкция: какие классы использовать и для чего.

    Пример гида (фрагмент)

    #### Button
- `.btn.btn--primary` — основное действие
- `.btn.btn--secondary` — вторичное действие
- `.btn.btn--ghost` — нейтральное действие

#### Card
- `.card` — контейнер
- `.card__title` — заголовок
- `.card__actions` — кнопки

#### Input
- `.field` — обёртка
- `.label` — подпись
- `.input` — поле
- `.help` — подсказка

    2. Задания

    Каждый студент работает в своей предметной области (сервис №1–30) и оформляет UI-kit под интерфейс этого сервиса.

    2.1. Подготовка структуры проекта

    Создайте структуру:

    lr3/
  index.html
  css/
    tokens.css
    components.css
    demo.css
  report/

    Требование: компоненты должны быть вынесены в отдельный файл components.css.

    2.2. Создание цветовой палитры (tokens)

    1. Определите палитру: primary, secondary, accent, neutral.
    2. Оформите токены в tokens.css через :root.

    Минимум: 10 переменных цветов.

    2.3. Типографическая шкала

    1. Настройте базовый шрифт.
    2. Задайте шкалу размеров в rem.
    3. Настройте line-height.

    Минимум: 5 размеров текста (--text-*) + 2 line-height.

    2.4. CSS Custom Properties (токены UI)

    Добавьте токены:

    • отступы (--space-*) минимум 6
    • радиусы (--radius-*) минимум 3
    • тени (--shadow-*) минимум 1

    2.5. Компонент Button

    Сделайте компонент .btn и модификаторы:

    • .btn--primary
    • .btn--secondary
    • .btn--ghost

    Состояния :

    • default
    • hover
    • active
    • focus-visible
    • disabled

    2.6. Компонент Card

    Сделайте .card + внутренние элементы (например):

    • .card__title
    • .card__text
    • .card__actions

    2.7. Компонент Input

    Сделайте:

    • .field, .label, .input, .help, .error

    Состояния :

    • focus-visible
    • invalid ([aria-invalid="true"])
    • disabled

    2.8. Компонент Alert

    Сделайте базовый .alert и варианты:

    • .alert--info
    • .alert--success (можно на secondary)
    • .alert--error (на accent)

    2.9. Hover / Focus / Active (глобальные правила)

    Добавьте глобальные правила (минимум):

    • видимый focus для интерактивных элементов
    • hover на ссылках
    • active на кнопках

    2.10. Темная тема (prefers-color-scheme)

    Реализуйте dark-mode в tokens.css:

    • переопределите --bg, --surface, --text, --border, --shadow-*.

    2.11. Демонстрационная страница компонентов

    В index.html сделайте страницу “UI Kit”, где показаны:

    • все варианты кнопок;
    • минимум 2 карточки;
    • минимум 3 поля ввода (включая invalid и disabled);
    • минимум 3 алерта (info/success/error);
    • пример “форма + карточка” как в выбранной предметной области.

    2.12. Мини-гайд

    В отчёте или в README.md:

    • перечислите компоненты и классы;
    • укажите, где менять токены (цвета/типографику).

    3. Контрольные вопросы

    1. Что делает prefers-color-scheme?
    2. Почему лучше использовать rem вместо px для типографики?
    3. Какие токены вы бы добавили для масштабирования дизайн-системы?
    4. Чем компонент отличается от набора случайных классов?
    5. Почему focus-visible важен для доступности?

    4. Чек-лист для самопроверки

    Баллы Критерии выполнения
    4 Полный UI-kit: токены, типографика, 4 компонента, состояния, dark mode, demo-страница, мини-гайд
    3 Почти всё выполнено, мелкие недочёты (например, не все состояния или слабый dark-mode)
    2 Есть токены и компоненты, но нет системы (часть значений “магические”), demo слабая
    1 Частичная реализация: 1–2 компонента, токены неполные
    0 Нет дизайн-системы/ только разрозненные стили/ нет работы/ выполнена через ИИ

    5. Для выполнения требуется

    1. Шаблон отчёта, указанный в описании лабораторной работы.
    2. Вариант задания, закреплённый за студентом на семестр и опубликованный в ЛМС.
    3. Требования к оформлению — согласно методическим указаниям в ЛМС или соответствующему разделу документации.

    6. Формат сдачи работы

    1. index.html
    2. css/tokens.css, css/components.css, css/demo.css
    3. Отчёт .docx или .pdf
    4. Ссылка на GitHub-репозиторий
    5. Архив LR3_Фамилия_Группа.zip

    7. Запрет

    ❗ Запрещается использование систем искусственного интеллекта для выполнения лабораторной работы.

    Работы, выполненные с использованием ИИ, полностью или частично скопированные, а также не сданные, оцениваются в 0 баллов без возможности доработки.
