    Лабораторная работа №1 — Применение тегов HTML при создании web-страниц

    Цель

    Создать статическую HTML-страницу по индивидуальной предметной области, применив расширенный набор HTML-элементов: семантика, контент-разметка, таблицы, медиа, встраивания, интерактивные элементы, расширенная форма, доступность, валидация, а также опубликовать результат в GitHub с предоставлением ссылки.

    1. Теория

    1.1. HTML-документ и роль разметки

    HTML (HyperText Markup Language) — язык разметки, предназначенный для описания структуры веб-страницы. HTML-документ состоит из разделов head (метаданные) и body (контент страницы). Корректная структура документа обеспечивает правильное отображение в браузере, удобство сопровождения и соответствие стандартам.

    HTML-документ делится на:

    • <head> — метаданные (кодировка, заголовок вкладки, описание, иконка, настройки отображения).
    • <body> — содержимое страницы, которое видит пользователь.

    1.2. Каркас документа и метаданные (head)

    <!DOCTYPE html>

    Назначение: включает стандартный режим HTML5 (без “quirks mode”). Пример: всегда первая строка файла.

    <!DOCTYPE html>

    <html lang="ru">

    Назначение: корневой элемент документа; lang помогает доступности и корректному выбору языка.

    <html lang="ru">

    <head> … </head>

    Назначение: контейнер метаданных.

    <head>
 ...
</head>

    <meta charset="utf-8">

    Назначение: кодировка, чтобы русский текст отображался корректно.

    <meta charset="utf-8">

    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    Назначение: корректное масштабирование на телефонах.

    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <title>

    Назначение: название вкладки браузера/закладки.

    <title>ServiceDesk — оформление заявки</title>

    <meta name="description" content="...">

    Назначение: краткое описание страницы (часто используется в сниппетах поиска).

    <meta name="description" content="Страница по предметной области ServiceDesk: услуги, регламент, таблица заявок и форма обращения.">

    <meta name="author" content="...">

    Назначение: автор страницы (метаданные).

    <meta name="author" content="Иванова А.А., ДБИ-281рсобф">

    <link rel="icon" href="...">

    Назначение: иконка сайта (favicon).

    <link rel="icon" href="assets/favicon.ico">

    Дополнительные (одно на выбор)

    <base href="...">

    Назначение: задаёт базовый URL для относительных ссылок (аккуратно: влияет на все ссылки).

    <base href="/site/">
    <meta name="keywords" content="...">

    Назначение: список ключевых слов.

    <meta name="keywords" content="ServiceDesk, заявки, поддержка, регламент, SLA">
    <meta name="theme-color" content="#...">

    Назначение: цвет интерфейса браузера на мобильных.

    <meta name="theme-color" content="#dc143c">

    1.3. Семантическая структура страницы (body)

    <header>

    Назначение: “шапка” страницы или раздела.

    <header id="top">
 <h1>ServiceDesk</h1>
 <p>Портал регистрации и обработки обращений пользователей.</p>
</header>

    <nav>

    Назначение: навигация по странице/сайту.

    <nav aria-label="Навигация по разделам">
 <ul>
 <li><a href="#about">О системе</a></li>
 <li><a href="#services">Услуги</a></li>
 <li><a href="#table">Таблица заявок</a></li>
 <li><a href="#form">Форма обращения</a></li>
 </ul>
</nav>

    <main id="main">

    Назначение: уникальное основное содержимое страницы (должен быть один main).

    <main id="main">
 <!-- основной контент -->
</main>

    <section>

    Назначение: логический раздел с тематикой.

    <section id="about">
 <h2>Описание предметной области</h2>
 <p>Система фиксирует обращения, назначает исполнителя и контролирует SLA.</p>
</section>

    <article>

    Назначение: самостоятельная единица контента.

    <article>
 <h3>Услуга: Сброс пароля</h3>
 <p>Оформляется пользователем через форму. Исполнитель: администратор домена.</p>
</article>

    <aside>

    Назначение: дополнительный контент.

    <aside>
 <h3>Быстрые подсказки</h3>
 <p><strong>SLA:</strong> критические заявки — до 4 часов.</p>
</aside>

    <footer>

    Назначение: подвал страницы.

    <footer>
 <p>© 2026 Учебный проект</p>
 <address>
 Контакты: <a href="mailto:support@example.com">support@example.com</a>
 </address>
</footer>

    2. Задание

    2.1. Создайте каркас HTML-документа и метаданные (head)

    Создайте файл index.html. В документе обязательно должны присутствовать:

    • <!DOCTYPE html>
    • <html lang="ru">
    • <head> … </head>
    • <meta charset="utf-8">
    • <meta name="viewport" content="width=device-width, initial-scale=1.0">
    • <title>…</title>
    • <meta name="description" content="...">
    • <meta name="author" content="...">
    • <link rel="icon" href="...">

    Дополнительно (выбрать 1 из 3):

    • <base href="..."> (если используете структуру папок и относительные ссылки), или
    • <meta name="keywords" content="...">, или
    • <meta name="theme-color" content="#...">

    2.2. Реализуйте семантическую структуру страницы (body)

    В body обязательно применить следующие теги:

    • <header> — шапка страницы (название + краткое описание предметной области)
    • <nav> — меню с внутренними якорями
    • <main id="main"> — основной контент
    • минимум 2× <section>
    • минимум 1× <article>
    • <aside> — боковой блок (быстрые факты/термины/подсказки)
    • <footer> — подвал

    Дополнительно обязательно применить:

    • <address> — контактные данные (в footer или отдельной секции)

    2.3. Текстовая разметка (контент по предметной области)

    Добавьте текст, связанный с предметной областью, применив:

    Обязательные теги:

    • заголовки: <h1>, <h2>, <h3> (минимум по 1)
    • абзацы: <p> (минимум 5)
    • разделители: <br> (минимум 1 по смыслу), <hr> (минимум 1)

    Смысловые и типографские теги (использовать все):

    • <strong>, <em>
    • <mark>, <small>
    • <sub>, <sup>
    • <abbr title="...">
    • <time datetime="YYYY-MM-DD">
    • <cite>, <q>, <blockquote>
    • <pre><code>…</code></pre>
    • <kbd>, <samp>, <var>
    • <span>, <div> (не вместо семантики, а точечно)

    2.4. Списки (все основные виды)

    Создайте:

    1. <ol> — нумерованный список (минимум 5 пунктов)
    2. <ul> — маркированный список (минимум 6 пунктов)
    3. вложенный список (минимум 2 уровня вложенности)
    4. список определений: <dl>, <dt>, <dd> (минимум 3 термина)

    2.5. Ссылки и навигация

    Добавьте:

    • внешнюю ссылку <a href="https://..."> с target="_blank" и rel="noopener noreferrer"
    • внутренние якоря на разделы (оглавление)
    • ссылку «Наверх» (#top)
    • mailto: ссылку (электронная почта)
    • tel: ссылку (телефон)

    2.6. Изображения и медиа

    Обязательно:

    • <img> минимум 2:

    • у всех alt

    • у одного loading="lazy"
    • <figure> + <figcaption> минимум 1 (подпись к изображению)
    • адаптивная картинка: <picture> + <source> + <img> (1 раз)

    Аудио/видео (1 из 2):

    • <audio controls> или
    • <video controls>

    Если файлов нет, допускается заменить этот пункт на <iframe> (см. 2.7).

    2.7. Встраивания и встроенный контент (минимум 2 элемента)

    Выполните минимум 2 пункта:

    • <iframe> (карта/видео/страница)
    • <embed> (встраивание PDF)
    • <object> (альтернативное встраивание)
    • <noscript> (сообщение пользователю)
    • <canvas> (заглушка с пояснением назначения)
    • <svg> (простая иконка/фигура)

    2.8. Таблица по предметной области (полный набор)

    Создайте таблицу минимум на 5 строк данных и 4 колонки, используя:

    • <table>, <caption>
    • <thead>, <tbody>, <tfoot>
    • <tr>, <th>, <td>
    • scope="col" и scope="row"

    Дополнительно (1 из 2):

    • объединение ячеек rowspan/colspan, или
    • итоговая строка в <tfoot> (сумма/итог/количество)

    2.9. Сложная форма (расширенная, “максимум типов”)

    Форма должна соответствовать предметной области: заявка / заказ / регистрация / обращение / бронирование.

    Структурные теги :

    • <form method="post" action="#">
    • <fieldset> ×2
    • <legend> ×2
    • <label> для каждого поля

    Требование по полям: использовать минимум 14 разных элементов/типов, включая:

    input (минимум 10 разных type):

    • text, email, password, number, tel, date, time, url, file, range
    • дополнительно (можно, если уместно): color, search, hidden

    выбор и текст:

    • <select> + <option> + <optgroup> (optgroup обязателен)
    • <textarea>

    переключатели:

    • radio (3+ вариантов)
    • checkbox (2+ чекбокса, один — согласие required)

    кнопки:

    • <button type="submit">
    • <button type="reset">

    Продвинутые элементы формы (выбрать 2 из 4):

    • <datalist>
    • <output>
    • <progress>
    • <meter>

    HTML-валидация :

    • required минимум на 6 полях
    • min/max минимум на 2 полях
    • minlength/maxlength минимум на 2 полях
    • pattern минимум на 2 полях (телефон/пароль/код)
    • autocomplete минимум на 3 полях
    • placeholder по смыслу

    2.10. HTML-интерактив без JavaScript

    Добавьте:

    • <details> + <summary> — минимум 2 блока (FAQ/правила/подсказки)
    • <dialog> — как демонстрация структуры (открытие без JS не обязательно)
